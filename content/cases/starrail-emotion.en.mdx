If Star Rail's business model is "emotional investment," the question narrows to one:<br/>
how is that emotion held in place through the UI?
<br/>
First, time.<br/>
The character detail screen and the growth and enhancement screens are designed with real depth.<br/>
There are many layers, and the information isn't consumed all at once, so dwell time grows naturally.<br/>
Especially in the stretches that call for understanding of and attachment to a character, there are many steps just to enter the UI.<br/>
It's an intended delay that says, "let's keep the player here a little longer."
<br/>
Second, imbalance.<br/>
Star Rail didn't unify all of its UI to a single tone.<br/>
Into the areas where you interact with characters, it folded the concept aggressively, while the rest of the UI stays refined like system UI.<br/>
This isn't a matter of design taste — it's the result of a decision about where to spend the player's emotion.
<br/>
Third, resources.<br/>
The gacha and character staging — the things the business model has to emphasize — receive overwhelming resources.<br/>
The UI of the repetitive, grind-it-out stretches, by contrast, is pared down to the extreme.<br/>
This allocation locks logically into the world.<br/>
So even as UI trends shift, this game's UI rules won't change easily.
<br/>
In the end, Star Rail's UI is strategy, not taste.<br/>
But this structure isn't the right answer for every game.
<br/>
If I were making a game,<br/>
I'd deliberately borrow the "pacing design" that stretches dwell time.<br/>
The character placement on the home screen, treated like a default answer, I'd strip away — unless it were a subculture game.
<br/>
UI isn't a pretty end product; it's an expression of where a game is trying to go.
<br/>
So I find myself asking again:<br/>
what is your UI reaching toward?
