게임을 읽으려 할 때<br/>
재미있다, 재미없다보다 중요한 질문이 있다.
<br/>
왜 이렇게 만들었지?
<br/>
물론 유저로서 좋아하고 싫어할 자유는 있다.<br/>
하지만 게임을 읽는 순간부터 조금 다른 시선이 필요해진다.
<br/>
어렵거나, 취향이 아니거나,<br/>
좋아하는 스타일의 디자인이 아니라는 이유로 게임을 플레이하지 않는 경우가 있다.<br/>
하지만 출시된 게임들은 세상에 공개된 답지에 가깝다.<br/>
플레이를 통해 생각보다 훨씬 많은 것을 배울 수 있다.
<br/>
나는 게임을 읽을 때 다섯 가지 질문을 던져본다.
<br/>
이 질문들은<br/>
게임이 유저에게 무엇을 느끼고 경험하도록 설계했는지 드러낸다.
<br/>
1.	유저가 어떤 감정을 느끼도록 설계됐는가?<br/>
긴장, 애착, 호기심, 두려움.<br/>
게임은 연출과 구조를 통해 감정을 만들어낸다.
<br/>
2.	유저를 언제 어떻게 빠져들게 만드는가?<br/>
몰입은 정보를 줄이거나, 시선을 통제하거나,<br/>
불편함을 남기는 방식으로도 설계된다.
<br/>
3.	유저의 선택은 언제 의미를 갖게 되는가?<br/>
모든 게임에는 선택지가 있다.<br/>
하지만 모든 선택이 의미 있는 것은 아니다.<br/>
선택은 결과가 반영될 때 비로소 무게를 갖기 시작한다.
<br/>
4.	유저는 언제 자신의 선택을 후회하게 되는가?<br/>
어떤 게임은 실패를 패널티로 만들고 어떤 게임은 실패를 학습으로 만든다.<br/>
책임 설계는 유저에게 묻는다.<br/>
왜 그렇게 플레이했는가?
<br/>
5.	유저는 언제 스스로 답을 찾게 되는가?<br/>
어떤 게임은 정보를 제공하고, 어떤 게임은 유저가 직접 발견하게 만든다.<br/>
답을 바로 주지 않을 때 플레이어는 장면을 더 오래 바라보게 된다.
<br/>
게임은 유저에게<br/>
무엇을 느끼게 할 것인가,<br/>
언제 몰입하게 할 것인가,<br/>
무엇에 책임을 지게 할 것인가,<br/>
무엇을 발견하게 할 것인가를 설계하는 체감 예술에 가깝다고 생각한다.
<br/>
시대log는 게임이 좋다 나쁘다를 평가하지 않는다.
대신 UI를 통해 무엇을 설계했는지 묻는다.<br/>
감정,<br/>
몰입,<br/>
선택,<br/>
책임,<br/>
관찰의 구조를 읽으며<br/>
디자인의 알맹이를 찾아가는 기록을 남긴다.
